── Layer 1: Signal Calculation ──────────────────────────────────
GEX Regime Score Pure Rules SpotGamma/FlashAlpha API → compare SPX spot vs gamma flip price → output positive/negative/magnitude. Identical quality. This is arithmetic. GEX is a calculated number. No model improves it. No improvement possible. Would add latency and opacity for no gain. No AI
VIX Term Structure Pure Rules Pull VIX M1, M2, M3 futures prices → calculate M1/M2 ratio → flag backwardation if ratio >1.0, steepness of inversion. Identical quality. Three numbers, one division. Fully deterministic. No improvement possible. No AI
DIX Dark Pool Index Pure Rules SqueezeMetrics publishes pre-calculated DIX daily. Pull number → compare vs 45% threshold → flag bullish/bearish/neutral. Identical quality. SqueezeMetrics already did the math. We just consume it. No improvement. Would be adding AI to interpret a single percentage. No AI
IV Rank (IVR) Calculation Pure Rules Pull 52-week IV high/low from Barchart → (current IV − low) / (high − low) × 100 → score 0–100. Identical quality. Standard formula. No model needed. Nothing to improve. Formula is exact. No AI
Dark Pool Cluster Detection Pure Rules Pull FINRA dark pool prints per ticker → group prints within ±0.5% price band → count clusters of 3+ prints → flag price level. Identical quality. Clustering is a simple grouping algorithm — no ML needed. Could use DBSCAN clustering — but standard bucketing works identically for this use case. No AI
Options Flow Sweep Detection Pure Rules Unusual Whales API flags sweeps. Filter: premium >$1M + ask-side + OTM + near-term expiry. Count per ticker per session. Identical quality. Filtering is rule-based. UW already classifies sweep vs block. Noise reduction — AI could de-duplicate hedge sweeps. Marginal improvement at V1. No AI
COT Positioning (Futures) Pure Rules Pull CFTC weekly COT → calculate managed money net long % → compare vs 52-week range → flag extreme long/short. Identical quality. Percentile ranking of a weekly number. No improvement. Deterministic percentile math. No AI
OVX / GVZ Term Structure Pure Rules CBOE publishes OVX (oil VIX) and GVZ (gold VIX). Same M1/M2 ratio logic as VIX. Inversion = risk premium. Normalization direction = exit signal. Identical quality. Same arithmetic as VIX term structure module. No improvement possible. No AI
── Layer 2: Scoring Engine ──────────────────────────────────────
Macro Regime Gate (0–10) Pure Rules 4 signals × fixed weights → sum → threshold. GREEN ≥6, YELLOW 4–5, RED ≤3. Weights set manually from signal research. Identical quality at V1. Fixed weights validated against historical events (Sep–Apr, Apr 8) before launch. V2: AI learns optimal weights from outcomes. V1 doesn't need this. No AI (V1)
Instrument EigenView Score (0–10) Pure Rules 5 signals × fixed weights per instrument type → sum → filter <6 → rank survivors by score descending. Identical quality at V1. Same logic as regime gate. Weights are research-validated, not learned. V2: ML weight optimization per instrument class. Meaningful improvement but not needed for launch. No AI (V1)
Setup Type Classification Pure Rules Decision tree: which signals fired → maps to setup type. GEX flip cross + IVR>70 + dark pool = REVERSAL. Contango restore + breadth healthy = CONTINUATION. Full checklist = POSITION. Identical quality. A decision tree with 4 output classes. Fully explainable. No ambiguity. ML classification would add opacity for no quality gain. Explainability matters here — traders need to trust the label. No AI
Entry / Stop Zone Calculation Pure Rules Entry = dark pool cluster price ± 0.5% OR GEX gamma flip level ± 0.3%. Stop = below nearest dark pool support / below gamma flip. Pure price math. Identical quality. These are specific price levels derived from the signal data. Arithmetic. No improvement. A model predicting entry zones from patterns would be less reliable than the mechanical levels. No AI
Timeframe Assignment Pure Rules Lookup table: setup type × signal combination → timeframe label. REVERSAL + GEX flip = Swing 3–10d. Full capitulation score = Position 2–6w. Static mapping. Identical quality. These ranges are derived from documented signal research, not learned. No value-add for a lookup table. No AI

── Layer 4: Where AI Actually Earns It ──────────────────────────
Catalyst Detection Real AI Monitor SEC 8-K filings, Fed statements, geopolitical news feeds in real time. Classify: does this event likely resolve the geopolitical risk premium on CL? Is this an unpriced positive catalyst for NVDA? Rules cannot do this. Unstructured text classification at scale is impossible with if/then logic. This is the April 8 problem — the ceasefire headline. A rules system has no way to parse it. AI is the only option. An LLM classifying news against instrument exposure in real time is a genuine moat. Detects catalyst before price moves. Real AI
Adaptive Signal Weight Learning Real AI After 6–12 months of signal log + outcomes, learn which signal combinations actually predict +4%+ moves in each instrument class. Adjust weights dynamically. Rules cannot improve themselves. Fixed weights are calibrated once from research. They don't update from new outcome data. ML (simple regression or gradient boosting) on outcome data genuinely improves score accuracy over time. This is the compounding moat — the product gets better the more it runs. Real AI
Cross-Instrument Correlation Alerts Hybrid Detect when unusual flow in one instrument historically precedes a move in a correlated instrument. E.g. XLE sweep before CL futures move, or semis ETF dark pool before NVDA print. Rules: can handle known static pairs (XLE→CL, SOXX→NVDA) with lookup tables. Covers 80% of cases. AI adds: dynamic correlation detection that finds non-obvious pairs from live data. Discovers new relationships rules would miss. Hybrid

The Honest Answer
EigenView at V1 is a rules-based signal aggregation and scoring engine. Roughly 70% of the product — signal calculation, scoring, ranking, entry/stop math, setup classification — is deterministic arithmetic that produces identical output for identical inputs every time. No AI, no hallucination risk, fully auditable.

That is a feature, not a limitation. Traders need to trust the output. A rules engine with a public signal log and verifiable win rate builds more trust than a black-box AI score. TradeAlgo uses AI scoring and explicitly cannot explain why it called something. EigenView can explain every score component because every component is a number derived from a published calculation.

AI earns its place at V2 in exactly two places: catalyst detection from unstructured news (the only way to catch April 8-type events in advance), and adaptive weight learning from outcome history (the compounding moat that makes the product better over time). Both require real outcome data to train on — which means V1 has to ship and run first anyway.

The correct build sequence: ship V1 as a pure rules engine. Collect 6 months of signal logs + outcomes. Then train the adaptive layer on real data. AI built on top of a validated rules foundation is far more accurate than AI built from scratch on raw market data.
