AI and UI are last — not first. A bad data pipeline with beautiful UI is useless. A working scoring engine with the wireframe's CSS is already the product.
Bottom Line
~70% of EigenView is pure rules-based Python math. GEX, TA, flow scoring, dormant-bet rule scoring, synthesis gate — all deterministic, zero API cost, zero hallucination risk. This is the engine. It should be built and tested before a single AI call is wired.

~20% is Claude API — and it's worth it for exactly 4 things: pick thesis generation, news MATERIAL/NOISE classification, dormant-bet narrative, and AI chat. These four are where template strings fail and Claude shines. Combined daily cost: ~$0.45. Monthly: ~$15.

~10% is the UI wiring work — the wireframe v2 is the design, already done, already pixel-perfect. The remaining UI work is extracting it into modules and wiring it to the FastAPI backend. This is not design work. It's plumbing.

The single highest-risk AI component is the options structure suggestion. Claude will confidently suggest structures with wrong R:R math. Never display AI-generated options math without an independent rules-based validation layer recalculating the numbers.

Start by testing the AI output quality manually in this chat today. Paste a factor JSON, use the prompt template from doc 11, see if the thesis quality matches the wireframe's canned example. That answer tells you whether to proceed as-designed or tune the prompts first.
